⚡ 快速启动 (3步)
========================================

步骤 1: 安装 Docker
   - 右键运行 install-docker.ps1 (管理员权限)
   - 或手动下载: https://www.docker.com/

步骤 2: 配置环境
   - 已有默认配置 .env.local
   - 可直接使用测试，无需修改

步骤 3: 启动项目
   - 双击运行 run.bat
   - 等待启动完成
   - 访问 http://localhost:3000

📁 项目文件说明
========================================

启动脚本:
  run.bat                  ⭐ 一键启动 (推荐)
  start-docker.bat         - Docker 启动脚本
  docker-start-all.bat     - 完整 Docker 部署
  install-docker.ps1       - Docker 自动安装

配置文件:
  .env.local               - 环境变量配置
  .env.example             - 环境变量模板
  docker-compose.yml       - Docker 编排配置
  Dockerfile               - Docker 镜像配置

测试脚本 (testRun/):
  test-setup.bat           - 环境测试
  test-database.bat        - 数据库测试
  test-build.bat           - 构建测试
  test-api.ps1             - API 测试
  init-data.bat            - 初始化示例数据

========================================
🔧 常见操作
========================================

启动开发服务器:
  > run.bat
  
或手动启动:
  > docker compose up -d postgres
  > npm run db:push
  > npm run dev

停止服务:
  > Ctrl + C (停止应用)
  > docker compose down (停止数据库)

查看日志:
  > docker compose logs -f postgres

初始化示例数据:
  > npx tsx testRun\init-sample-data.ts

运行测试:
  > testRun\test-setup.bat

========================================
❓ 遇到问题？
========================================

问题1: Docker 未安装
解决: 运行 install-docker.ps1 或手动安装

问题2: 端口被占用
解决: 
  > netstat -ano | findstr :3000
  > taskkill /PID <进程ID> /F

问题3: 数据库连接失败
解决:
  > docker compose restart postgres
  > 等待10秒后重试

问题4: 构建失败
解决:
  > npm install
  > npm run db:generate
  > npm run build

更多帮助请查看:
  - INSTALL.md (详细故障排除)
  - README.md (常见问题FAQ)
